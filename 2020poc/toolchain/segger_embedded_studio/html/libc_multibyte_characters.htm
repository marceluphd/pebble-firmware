SEGGER Embedded Studio for ARM supports multi-byte encoding and decoding of characters. Most new software on the desktop uses Unicode internally and UTF-8 as the external, on-disk encoding for files and for transport over 8-bit mediums such as network connections.

However, in embedded software there is still a case to use code pages, such as ISO-Latin1, to reduce the footprint of an application whilst also providing extra characters that do not form part of the ASCII character set.

The SEGGER Embedded Studio for ARM C library can support both models and you can choose a combination of models, dependent upon locale, or construct a custom locale.
